API Usage

The API allows users to analyze speech data provided in MP3 format.

Input Values

Users are required to send the following data to the API:

  • file: The MP3 file containing the speech data to be analyzed.
  • api_key: The API key generated by the user for authentication purposes.
  • expected_sentiments: A JSON object containing the expected emotional sentiments desired in the speech. The object should follow this format:
{"optimism": 0.6, "excitement": 0.4, "caring": 0.4}

This example indicates that the user ideally wants the speech to be 60% optimistic, with bonus points to being close to 40% excited and/or 40% caring

Return Values

Upon sending the MP3 file and expected sentiments, the API returns the following information:

  • sentiment_score: A score out of 10, indicating the overall quality of the speech in terms of meeting the expected sentiments. Example: 8.4
  • pace_score: A score out of 10, indicating the overall pace or speed of the speech. Higher scores suggest a faster pace, while lower scores suggest a slower pace. Example: 5.7
  • diction_score: A score out of 10, indicating the overall quality of the word choice and vocabulary used in the speech. Higher scores suggest more effective and appropriate use of language, while lower scores suggest poorer word choice and bad grammar. Example: 4.5
  • total_score: A score out of 10, indicating the overall quality of the speech in terms of overall speech. Example: 9.0
  • tone_data: An array of tone data points indicating the pitch variations in the speech. Example: [0, 60, 50, ...]
  • tone_times: An array of tone times indicating the time stamps for the pitch variations in the speech. Example: [0, 0.25, 0.5, ...]
  • user_sentiments (SamLowe's Model): An array of objects representing the actual emotional content of the speech. Each object contains a label indicating the sentiment label (e.g., "caring") and a score indicating the sentiment score ranging from 0 to 1. Example:
    •  [
                {
                    "label": "caring",
                    "score": 0.24102665483951569
                },
                {
                    "label": "optimism",
                    "score": 0.654987123456789
                },
                ...
]
  • user_speech: A transcription of the user's speech, providing a textual representation of what was said. Example: "John seriously stop whining about your academic pressure..."

Example Code

Below is an example of how to call the API using JavaScript:


const apiKey = 'YOUR_API_KEY';

// Replace 'expectedSentiments' with your expected sentiments dictionary
const expectedSentiments = {
    "positive": 0.5,
    "negative": 0.3,
    "neutral": 0.2
};

// Function to send the POST request
async function sendRequest(mp3File) {
    // Create a FormData object to send multipart/form-data
    const formData = new FormData();
    formData.append('file', mp3File);
    formData.append('api_key', apiKey);
    formData.append('expected_sentiments', JSON.stringify(expectedSentiments));

    try {
        const response = await fetch('http://localhost:5000/api/analyze_speech', {
            method: 'POST',
            body: formData
        });

        // Check if the response is successful (status code 200-299)
        if (response.ok) {
            const data = await response.json();
            console.log(data); // Do something with the response data
        } else {
            console.error('Failed to fetch:', response.statusText);
        }
    } catch (error) {
        console.error('Error:', error);
    }
}

// Function to handle file selection
function handleFile() {
    const fileInput = document.getElementById('file-input');
    const file = fileInput.files[0]; // Get the selected file
    sendRequest(file); // Send the file to the server
}

To use this code, replace 'YOUR_API_KEY' with your actual API key and adjust the expectedSentiments object according to your desired sentiments. Call the handleFile function when you want to send a file to the API.
